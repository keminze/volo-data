# report-writer：商业洞察报告撰写技能

## 概述

本技能指导 Agent 生成高质量的商业洞察报告，不仅描述数据，更要透过数据看业务本质。

## 报告结构模板

### 【核心结论】
开门见山，用 1-2 句话直接回答用户问题。例：
> "过去30天总销售额为 ¥2,847,392，环比上月下降 12.3%，主要拖累来自华南区。"

### 【关键发现】
- 列出 3-5 条数据支撑的具体发现
- 每条发现配具体数字
- 区分"正面"和"需关注"的信号

### 【深度解读】
- 结合 DDL 中的业务逻辑解释异常数据
- 指出趋势转折点和背后可能的原因
- 对比历史（如有对话历史可参考）

### 【行动建议】
- 提出 1-2 个可落地的具体建议
- 建议应与数据直接关联，不空泛

## 调用 generate_analysis_report

传入参数：
- `user_question`：用户原始问题
- `data`：最终数据（优先用 `code_result`，否则用 `execute_sql_query` 的 `data`）
- `sql`：执行的 SQL 语句
- `ddl`：相关表 DDL
- `user_history`：近期对话历史（用于理解上下文）

## 写作规范

- 使用"我注意到..."、"从数据来看..."等自然语气
- 数值精确到 3 位小数
- 严禁使用数据库字段名（`order_id` → "订单ID"）
- 严禁自行换算单位
- 空数据时回复："抱歉，没有找到相关数据。"（30字内）
