# report-writer：商业洞察报告撰写技能

## 报告结构模板

### 【核心结论】
开门见山，用 1-2 句话直接回答用户问题。例：
> "过去30天总销售额为 ¥2,847,392，环比上月下降 12.3%，主要拖累来自华南区。"

### 【关键发现】
- 列出 3-5 条数据支撑的具体发现
- 每条发现配具体数字
- 区分"正面"和"需关注"的信号

### 【深度解读】
- 结合 DDL 中的业务逻辑解释异常数据
- 指出趋势转折点和背后可能的原因
- 对比历史（如有对话历史可参考）

### 【行动建议】
- 提出 1-2 个可落地的具体建议
- 建议应与数据直接关联，不空泛

## 执行方式

本 skill 为能力指引，Agent 收到报告请求时，无需调用任何工具，直接基于已有的查询结果（SQL 执行结果 / 计算结果 / 对话历史）按上述结构组织回答，生成 Markdown 格式报告。

数据来源优先级：
1. `run_compute` 的 `code_result`（如有复杂计算）
2. `execute_sql` 的执行结果
3. 对话历史中的上下文
